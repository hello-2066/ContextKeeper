ContextKeeper (会話の永続化)

再起動後の「記憶喪失」を防ぐ！LangBotに長期的かつ継続的な会話メモリを提供します。

ContextKeeperは、LangBot専用に設計されたコンテキスト管理プラグインです。コンテナの再起動、パイプラインのリロード、または設定更新後に会話コンテキストが失われる問題を解決します。

⚠️ プラットフォーム互換性：このプラグインは現在、WeChat Work (企業微信) スマートボットでのみ検証されています。

✨ 主な機能

1.  自動永続化
   会話のたびに、ユーザーの質問とAIの応答（Q&A）をローカルストレージに自動的に保存します。

2.  シームレスな復元
   LangBotの再起動やパイプラインのリロードに関わらず、過去の履歴を自動的に読み込みます。

3.  スマートコンテキスト注入
   トークン節約: LangBotのメモリ内にすでにネイティブコンテキストが存在する場合（会話開始直後など）、注入を自動的にスキップし、トークンの無駄遣いを防ぎます。
   自動補完: 「記憶喪失」状態が検出された場合にのみ、直近の 20ターン（設定可能）の会話履歴をプロンプトに注入します。

4.  コマンド制御
   ユーザーはコマンドを使用していつでも記憶を消去し、新しいトピックを開始できます。

 使用可能なコマンド

チャットで以下のキーワードのいずれかを送信すると、現在のユーザーの履歴メモリが消去されます：

*   `/clear`
*   `重置对话`
*   `重置会话`
*   `清除记忆`
*   `清空对话`
