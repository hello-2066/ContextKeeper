ContextKeeper (会话持久化)

拒绝重启“失忆”！让你的 LangBot 拥有长久、连续的对话记忆。

ContextKeeper 是一个专为 LangBot 设计的上下文管理插件。它解决了 LangBot 在容器重启、流水线重载或更新配置后，会话上下文丢失导致机器人“失忆”的问题。

⚠️ 平台兼容性：本插件目前仅在 企业微信 (WeChat Work) 智能机器人 上进行了完整验证。

✨ 核心功能

1.  自动记忆持久化
   在每一轮对话结束后，插件会自动将用户的提问和 AI 的回复（Q&A）持久化存储到本地磁盘/数据库中。

2.  重启无缝恢复
   无论是因为 LangBot 重启还是流水线重载，插件都能自动读取之前的历史记录并注入上下文。

3.  智能上下文注入
   节省 Token：如果检测到 LangBot 内存中已有原生上下文（如刚启动或已有记忆），插件会自动跳过注入，避免重复发送 Prompt，防止 Token 翻倍浪费。
   自动补全：仅在检测到“失忆”状态下，自动将最近的 20 轮（默认配置）对话历史注入到 Prompt 中。

4.  指令控制
   支持用户随时通过指令清除记忆，开启新的话题。

使用指令
在对话框中发送以下任一关键词，即可清除当前用户的历史记忆：

*   `重置对话`
*   `重置会话`
*   `/clear`
*   `清除记忆`
*   `清空对话`
